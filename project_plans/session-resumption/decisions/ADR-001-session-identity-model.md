# ADR-001: Session Identity Model for Resumption

**Status**: Accepted
**Date**: 2026-04-03
**Context**: Session Resumption feature — how to stably identify sessions across restarts, PID reuse, and cold restores.

## Decision

Use `claude_conversation_uuid` (the filename of Claude's JSONL conversation file) as the **primary stable session identity** for resumption purposes. Augment with a composite `ProcessFingerprint` for liveness checking.

### Identity Hierarchy

```
Layer 1 (Stable — survives restarts, reboots):
  claude_conversation_uuid  <- from ~/.claude/projects/<hash>/<UUID>

Layer 2 (Session-scoped — survives restarts):
  claude_mux_socket_path    <- /tmp/claude-mux-<PID>.sock

Layer 3 (Ephemeral — changes on restart):
  {pid, pid_start_time_ms}  <- for liveness disambiguation
```

### Resolution Order on Restart

1. Load persisted `claude_conversation_uuid` from session storage
2. Check if a tmux session with the matching name is alive (hot resume)
3. If not, check if a `claude-mux` socket exists for this session
4. If neither, cold restore: launch `claude --resume <uuid>` in saved cwd

## Alternatives Considered

1. **PID-only identity**: Rejected. PIDs are reused aggressively on macOS (max 99,998). A PID alone becomes ambiguous within minutes of a process exiting.

2. **Tmux session name as identity**: Rejected as primary. Tmux session names are derived from the stapler-squad instance title, which can be renamed. The conversation UUID is immutable.

3. **Random UUID generated by stapler-squad**: Rejected. Claude already provides a stable UUID per conversation. Using our own UUID adds a mapping layer with no benefit.

## Consequences

- `ClaudeSessionData.SessionID` already exists and stores the conversation UUID — reuse it
- New field `HistoryFilePath` on `InstanceData` to persist the full path to the JSONL file
- PID-based liveness checks must always verify `gopsutil Process.CreateTime()` to guard against PID reuse
- History file detection must populate `ClaudeSessionData.SessionID` when it discovers the JSONL file

## Risks

- If the JSONL file is deleted by the user, the stable identity is lost. Mitigation: persist the UUID in stapler-squad's own session storage. The UUID is the recovery key even if the source file is gone.
- For non-Claude programs (Aider), there is no equivalent stable UUID. Mitigation: fall back to cwd + program name as the identity composite. Aider resume is implicit (same directory).
